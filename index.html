Quick Configuration

Set up Hutrol in seconds using either cloud models or local air-gapped models.

Option 1: OpenRouter (Cloud)

# Set the provider
hutrol config set PROVIDER openrouter

# Add your API Key
hutrol config set OPENROUTER_API_KEY sk-or-...

# Choose a Tool-Calling model
hutrol config set OPENROUTER_MODEL meta-llama/llama-3.1-8b-instruct

Option 2: Ollama (Air-Gapped)

# Switch to local provider
hutrol config set PROVIDER ollama

# Set your local model name
hutrol config set OLLAMA_MODEL llama3.1

# Set the host URL
hutrol config set OLLAMA_HOST http://localhost:11434

How to Use Hutrol

A completely natural language interface for your operating system.

1. Interactive REPL Mode

Start an ongoing conversation with the agent. It retains context across queries, allowing you to ask follow-up questions while the state machine remembers everything.

hutrol repl

2. Single Command Execution

Run a specific task and exit immediately. Ideal for CI/CD pipelines, automated scripts, or fast one-off requests from the terminal.

hutrol run "Find all python files and tell me how many there are"

3. Command Risk Rules

Set the safety risk classification for a system command dynamically. Red completely blocks the command, Yellow prompts for [y/N] approval, and Green executes instantly.

hutrol rule set yellow Stop-Process

4. Compliance Export

Package your immutable system logs, tool execution traces, and human-in-the-loop decisions into a timestamped archive with a cryptographic SHA-256 Checksum.

hutrol export-audit
